fix(scrum-sim): players choose tasks themselves, no auto-pull into «In progress»

Sprint backlog → In progress is now an explicit player action:
- new POST /task/start and /task/return endpoints
- allocation no longer auto-promotes a backlog task to in_progress;
  capacity goes to tasks the team has already explicitly started
- daily UI splits the right column into two zones:
  * «Что можно начать сегодня» with ▶ Start buttons (and locked rows
    showing missing dependencies)
  * «Сейчас в работе» allocation list with an inline ↩ return action
    while the task has zero progress
- start endpoint rejects when deps aren't ready and returns the
  missing dep titles so the UI can show a clear hint

Verified with an in-memory smoke test: pull → confirm → no auto-promotion,
explicit start, return-while-zero-progress, blocked-by-deps rejection,
allocation drops sneaky backlog entries, end day only progresses
the explicitly started tasks.
